# contextd decision

Manage architecture decision records (ADRs) in your project.

## Usage

```bash
contextd decision <action> [title]
```

## Actions

### `add`

Create a new ADR with a sequential number and pre-filled template:

```bash
contextd decision add "Why we use tRPC instead of REST"
```

This creates a file in `.context/decisions/` with the next sequential number (e.g., `004-why-we-use-trpc-instead-of-rest.md`):

```markdown
---
title: "Why we use tRPC instead of REST"
date: 2026-05-03
status: accepted
tags: []
---

# ADR-004: Why we use tRPC instead of REST

## Context
What is the issue that we're seeing that is motivating this decision or change?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult because of this change?

## Alternatives Considered
What other options did we evaluate?
```

### `list`

List all existing ADRs with their numbers, titles, and statuses:

```bash
contextd decision list
```

Output is color-coded by status:
- Green: `accepted`
- Red: `deprecated`
- Yellow: other statuses (e.g., `proposed`)
